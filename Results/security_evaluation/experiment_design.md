# Result: Standalone Security Evaluation

## Objective

Evaluate the security posture of contracts produced by the proposed pipeline using Stage 3 analysis outputs.

This experiment isolates security as a primary result rather than treating it only as a supporting metric in other comparisons.

## Scope

The experiment uses contracts generated by the pipeline and their corresponding `stage3_report.json` files.

Recommended source:

- `pipeline_outputs/<run_id>/stage3_report.json`

Optional extension:

- include both initial generated contracts and final auto-fixed contracts when available

## Research Questions

1. How many security findings are detected in pipeline-generated contracts?
2. What is the severity distribution of these findings?
3. Which vulnerability classes appear most frequently?
4. How much improvement is achieved after the Stage 3 fixing loop, if auto-fix is enabled?

## Evaluation Metrics

### Security Findings

- total findings per contract
- average findings per contract
- percentage of contracts with at least one finding

### Severity Distribution

- critical findings
- high findings
- medium findings
- low findings
- informational findings

### Vulnerability Categories

- reentrancy
- access control
- arithmetic issues
- denial of service
- uninitialized storage
- unsafe external calls
- style or best-practice warnings

### Fixing Effectiveness

- initial issue count
- final issue count
- issues resolved
- fix rate percentage
- number of Stage 3 iterations used

## Dataset

Use the set of generated contracts in `pipeline_outputs/` covering multiple contract categories.

Recommended minimum:

- NFT
- token
- governance
- marketplace
- escrow
- crowdfunding
- auction
- timelock

## Procedure

1. Collect all available `stage3_report.json` files from pipeline runs.
2. Extract initial and final issue counts for each contract.
3. Group findings by severity and vulnerability type.
4. Compute aggregate statistics across all contracts.
5. Summarize the most frequent and most severe weaknesses.

## Expected Outputs

- `security_results.json` for per-contract metrics
- `security_summary.json` for aggregate counts
- `plots/severity_distribution.png`
- `plots/vulnerability_type_frequency.png`
- `plots/fix_effectiveness.png`
